Understanding CSS

Cascading Style Sheets - transforming plain HTML into beautiful designs

What is CSS?

CSS (Cascading Style Sheets) is used to style and layout web pages — for example, to alter the font, color, size, and spacing of your content, split it into multiple columns, or add animations and other decorative features.

While HTML provides the structure and content, CSS brings it to life with colors, layouts, and visual interactions.

Core CSS Concepts

CSS is based on selecting elements and applying properties. The cascade and specificity determine which rules apply when multiple styles target the same element.

  • Selectors: Target HTML elements (element, class, ID, attribute selectors). Use specific selectors to apply styles where needed.
  • Properties: Define what to style (color, size, position, etc.) and can be mixed into themes.
  • Values: Specify how to style (px, em, %, colors, etc.), including relative units for responsiveness.
  • Box Model: Understanding margin, padding, border, and content - the foundation of layout and spacing.
  • Flexbox & Grid: Modern layout techniques for responsive design and consistent spacing.
  • Responsive Design: Making layouts adapt to different screen sizes using media queries and relative units.

CSS Syntax & Selectors

Basic CSS Structure

selector {
  property: value;
  property: value;
}

Example: Styling Different Elements

/* Element selector */
p { color: blue; }

/* Class selector */
.highlight { background: yellow; }

/* ID selector */
#header { font-size: 2rem; }

/* Combining selectors */
nav a:hover { color: orange; }

CSS Variables (Custom Properties)

CSS Variables allow you to store reusable values and change them dynamically:

:root {
  --primary-color: #163a76;
  --spacing-unit: 8px;
  --transition-speed: 0.3s;
}

.button {
  background: var(--primary-color);
  padding: calc(var(--spacing-unit) * 2);
  transition: background var(--transition-speed) ease;
}

How it works: Custom properties are defined with --name and accessed using var(--name). You can override them in specific sections (like a dark mode theme) and provide fallbacks: var(--color, black).

.dark-mode { --primary-color: #0b1d3a; --surface: #112; }

Media Queries for Responsive Design

Media queries allow different styles for different screen sizes:

/* Mobile first - base styles for small screens */
.container { width: 100%; }

/* Tablet and up */
@media (min-width: 768px) {
  .container { width: 90%; }
  .grid { grid-template-columns: 1fr 1fr; }
}

/* Desktop and up */
@media (min-width: 1024px) {
  .container { width: 1120px; }
  .grid { grid-template-columns: 1fr 1fr 1fr; }
}

Advanced CSS Selectors

CSS offers powerful selectors for precise targeting:

/* Pseudo-classes */
a:hover { color: blue; } /* On hover */
input:focus { outline: blue; } /* When focused */
li:first-child { margin: 0; } /* First element */
li:last-child { border: none; } /* Last element */

/* Pseudo-elements */
p::first-line { font-weight: bold; } /* First line of paragraph */
input::placeholder { color: gray; } /* Placeholder text */
li::before { content: "→"; } /* Add content before element */

/* Attribute selectors */
input[type="email"] { border: 2px solid blue; }
a[href^="https"] { color: green; } /* Links starting with https */

CSS Performance Best Practices

  • Specificity: Use low specificity selectors for easier overrides
  • Avoid inline styles: Keep CSS separate from HTML for maintainability
  • Organize with comments: Group related styles, use meaningful names
  • Minimize repaints: Use transform and opacity for animations (GPU-accelerated)
  • Use shorthand: margin: 10px 20px instead of multiple rules
  • Mobile-first: Start with mobile, enhance for larger screens
