Understanding HTML

HyperText Markup Language - the building block of all web pages

What is HTML?

HTML (HyperText Markup Language) is the standard markup language used to create web pages. It provides the structure and content for websites by using a system of tags and elements that tell browsers how to display information.

Think of HTML as the skeleton of a webpage - it defines the basic structure, but CSS adds the styling and JavaScript adds the interactivity.

When a browser loads a page, it parses the HTML into a Document Object Model (DOM) tree. JavaScript can then read and modify that DOM, which is how dynamic web apps are built.

Key HTML Concepts

Understanding the building blocks of HTML helps you write clear, maintainable code. Semantic tags give meaning to content, which improves accessibility and makes your pages easier to maintain.

  • Tags and Elements: HTML uses paired tags like <tag> and </tag>.
    Tags are the markup symbols (like <p> for paragraph), and elements are the complete structure including opening tag, content, and closing tag. For example, <p>Hello World</p> is a complete paragraph element.
  • Semantic HTML: Using meaningful tags like <header>, <nav>, <main>, <footer> to describe the role of content.
    Instead of generic <div> tags everywhere, semantic elements tell browsers and screen readers what each section does - <header> for site branding, <nav> for menus, <main> for primary content, <article> for independent content like blog posts.
  • Attributes: Adding properties to tags, like href in <a href="url">, or alt in <img>.
    Attributes provide extra information about elements. Common ones include id (unique identifier), class (styling group), src (resource location), alt (image description for accessibility), and data-* (custom data storage).
  • Forms: Creating interactive input elements for user data collection and submission.
    Forms use elements like <input>, <textarea>, <select>, and <button> to gather user input. They submit data to servers via GET or POST methods, enabling user interactions like logins, searches, and data entry.
  • Accessibility: Ensuring content is available to all users, including those with disabilities.
    This means adding alt text to images, using proper heading hierarchy (h1-h6), ensuring keyboard navigation works, providing sufficient color contrast, and using ARIA attributes when needed for complex interactions.

Common HTML Elements

HTML elements form the content of your page. Using the right element for the right purpose ensures better readability, accessibility, and maintainability.

Structural Elements

<!DOCTYPE html> <!-- Document type declaration -->
<html>
  <head>...</head> <!-- Meta information -->
  <body>
    <header></header> <!-- Site header -->
    <nav></nav> <!-- Navigation menu -->
    <main></main> <!-- Main content -->
    <footer></footer> <!-- Site footer -->
  </body>
</html>

Content Elements

These elements are used to build the actual content on the page. Proper use makes the page easier for users and search engines to understand.

  • <h1> to <h6> - Headings (h1 most important, h6 least)
  • <p> - Paragraphs of text
  • <ul> / <ol> - Unordered (bullets) and ordered (numbered) lists
  • <a href=""> - Links to other pages or resources
  • <img src="" alt=""> - Images with alternative text for accessibility
  • <strong> / <em> - Bold and italic emphasis (semantically meaningful)

Demo: HTML Form

<form>
  <label for="name">Full Name</label>
  <input type="text" id="name" name="name" placeholder="Enter your full name" required>

  <label for="email">Email Address</label>

  <label for="message">Message</label>
  <textarea id="message" name="message" placeholder="Tell us about yourself..."></textarea>

  <button type="submit">Submit Form</button>

Elements shown: text input, email input, select dropdown, textarea, and submit button.

How it works: The <form> element wraps inputs; <label> associates with inputs via for attribute; name attributes send data to server; required prevents submission without input; type="email" validates email format.

HTML5 Modern Features

HTML5 introduced many new elements and attributes that improve semantics and functionality:

These additions make it easier to build rich, accessible pages without relying on extra libraries or complicated workarounds. They help browsers and search engines better understand your content.

  • <article>, <section>, <nav>: Semantic elements that replace generic divs
  • <figure> & <figcaption>: Meaningful image grouping with captions
  • <video> & <audio>: Native media support without plugins
  • data-* attributes: Store custom data on elements
  • Input types: email, date, number, range, color with built-in validation
  • required, pattern, placeholder: Attributes that improve UX and validation

HTML Document Structure Best Practices

A well-structured HTML document ensures your page is accessible, searchable, and easy to maintain. The head section contains metadata that improves performance and search engine visibility.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Page description for SEO">
  <title>Descriptive Page Title</title>
</head>
<body>
  <header></header> <!-- Site header/navigation -->
  <main> <!-- Main content -->
    <article></article>
  </main>
  <footer></footer> <!-- Site footer -->
</body>
</html>

Why this matters: Semantic HTML improves SEO, accessibility, and maintainability. Setting the correct language, viewport, and meta description helps search engines and assistive technology interpret your site correctly.
